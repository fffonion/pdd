# 拼豆豆 图纸转换

在线演示：https://yooooo.us/beads

一个基于 Bun、React、TypeScript、Tailwind CSS 和 Radix UI 的前端工具，用来把图片转换成拼豆图纸，并在网页里继续编辑、查看和导出。

![工作流预览](images/work.jpg)

![编辑器预览](images/edit.jpg)

## 项目特点

- 全部处理都在浏览器内完成，不依赖后端服务
- 支持自动识别像素画，也支持手动指定网格
- 支持手动裁切，并在原图里显示当前生效的 crop 框
- 支持普通图片先像素化，再匹配拼豆色系
- 支持多种拼豆色系，不只限于 MARD
- 支持颜色归并、低频杂色吸附、预锐化
- 支持像素级编辑、撤销重做、填充、吸管、橡皮、画笔
- 支持拼豆模式高亮单色、全屏查看、缩放、拖拽平移、触屏手势
- 支持禁用某个匹配色，并自动替换为当前结果中最接近的已使用颜色
- 支持中文和英文界面，以及浅色、深色、跟随系统主题

## 本地运行

安装依赖：

```bash
bun install
```

启动开发环境：

```bash
bun run dev
```

生产构建：

```bash
bun run build
```

本地预览构建结果：

```bash
bun run preview
```

运行测试：

```bash
bun test
```

## 非根路径部署

构建时可通过 `PINDOU_BASE_PATH` 指定基础路径：

```bash
PINDOU_BASE_PATH=/your/sub/path bun run build
```

未指定时默认使用相对路径，适合直接部署到子目录。

## 使用流程

1. 打开页面后，先在 landing page 选择图片。
2. 上传后进入完整工作区。
3. 在左侧设置网格策略、裁切、图像整理参数和色系。
4. 在右侧使用“像素编辑”或“拼豆模式”继续处理结果。
5. 最后下载生成的图纸 PNG。

## 主要功能

### 1. 网格识别

- 自动模式会优先尝试识别已有网格的像素画
- 自动识别失败时，会切换到手动网格模式
- 手动网格支持“跟随图片比例”
- 手动裁切后，会自动切到手动网格模式
- 对明显不合理的自动识别结果，会回退到手动模式

### 2. 图像整理

- 颜色归并：合并视觉上接近的颜色，减少逻辑颜色数
- 低频色吸附：把只出现 1 到 2 次、且与周围主色足够接近的颜色并入周围主色
- 预锐化：用于普通图片转像素图前，减少模糊边缘
- 普通图片转像素图时，会按逻辑网格采样，不直接依赖模糊缩放

### 3. 色系匹配

- 默认支持 `MARD 221`
- 也支持 `MARD Full` 与其它拼豆色系映射
- 内部颜色相似度使用 `OKLab`
- 背景不会简单按某个色号硬编码，而是按开放连续区域识别

### 4. 像素编辑

- 支持平移、缩放、画笔、橡皮、吸管、填充
- 画笔和橡皮的历史记录按一次按下到抬起记为一步
- 支持蜂巢式调色器，并可按色号过滤
- 支持在编辑模式里水平翻转
- 触屏下支持平移和 pinch 缩放

### 5. 拼豆模式

- 显示不可编辑的图纸视图
- 点击底部颜色或图纸格子可高亮该颜色
- 高亮某个颜色时，其它颜色会淡化，且其它格子的文字会隐藏
- 支持每 5 格虚线、每 10 格实线，以及 X/Y 轴编号
- 支持水平翻转、滚轮缩放、按钮缩放、拖拽平移、触屏缩放和平移
- 支持整页专注模式，只显示图纸和底部颜色区

### 6. 结果与导出

- 可禁用某个匹配色，并自动替换成当前结果里最接近的已使用颜色
- 颜色禁用状态会实时反映到画布和导出结果
- 导出文件名默认是 `【拼豆豆】原始文件名.png`
- 导出的 PNG 包含品牌标题、logo、图纸标题、色系信息、颗粒总数和底部色卡统计

## 当前布局

- 顶部：品牌标题、主题切换、语言切换
- 左侧：上传图片、原图预览与裁切、网格策略、图像整理、色系选择
- 右侧：像素编辑 / 拼豆模式 选项卡与下载图纸按钮

## 数据来源

- `src/data/mard-palette-221.json`
- `src/data/color-system-mapping.json`

## 当前适用场景

- 已有网格的像素图
- 放大后的像素图
- 带分隔线或格缝的像素图
- 普通图片转拼豆图纸
- 一部分第三方软件导出的图纸截图

## 已知限制

- 自动识别并不能覆盖所有输入图片
- 某些第三方导出的图纸截图，目前自动 crop 仍在持续收敛
- 复杂图片通常仍建议结合手动网格和裁切一起使用
- 浏览器端图像处理会占用一定内存，超大图片可能较慢

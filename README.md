# 拼豆图纸转换

一个基于 Bun、React、TypeScript、Tailwind 和 Radix UI 的前端工具，用来把图片转换成拼豆图纸，并支持在网页里继续编辑结果。

在线演示：https://yooooo.us/beads

## 项目特点

- 全部处理都在浏览器内完成，不依赖后端服务
- 支持自动识别像素画网格
- 支持手动指定网格，并按图片比例联动宽高
- 支持普通图片转像素图后再匹配拼豆色系
- 支持多种拼豆色系，不只限于 MARD
- 支持颜色归并、预锐化、结果导出
- 支持像素级编辑
- 支持画笔、橡皮、吸管、填充、撤销、重做
- 支持禁用匹配颜色，并自动替换为当前结果中最接近的已使用颜色
- 支持中英文界面和浅色、深色、跟随系统主题

## 本地运行

先安装依赖：

```bash
bun install
```

启动开发环境：

```bash
bun run dev
```

生产构建：

```bash
bun run build
```

如需部署到非根路径，可在构建前指定基础路径：

```bash
PINDOU_BASE_PATH=/your/sub/path bun run build
```

不指定时会默认使用相对路径构建，适合直接部署到子目录。

本地预览构建结果：

```bash
bun run preview
```

## 使用流程

1. 上传一张图片
2. 选择网格策略
3. 需要时裁切原图
4. 选择拼豆色系
5. 调整降色、容差、预锐化等参数
6. 在右侧像素编辑区继续修图
7. 下载生成的 PNG 图纸

## 主要功能说明

### 1. 网格识别

- 自动模式会优先尝试把输入识别成已有网格的像素画
- 如果自动识别失败，会切到手动模式
- 手动模式下可输入网格宽高，并可跟随图片比例自动联动

### 2. 图像整理

- 颜色归并：合并视觉上接近的颜色，减少过碎的逻辑颜色
- 低频色吸附：出现次数很少的颜色会优先向周围相近主色靠拢
- 预锐化：用于普通图片转像素图前，减少模糊边缘

### 3. 色系匹配

- 默认支持 `MARD 221`
- 也支持其他拼豆色系
- 颜色相似度计算内部使用 `OKLab`
- 纯白背景会作为空白处理，不计入颜色统计

### 4. 像素编辑

- 画笔：按当前颜色绘制
- 橡皮：移除像素
- 吸管：从当前画布取色
- 填充：按阈值批量替换相邻区域
- 原图叠加：把原图覆盖到编辑画布上辅助修图

### 5. 结果区

- 显示网格大小和逻辑颜色数
- 显示匹配到的颜色及数量
- 可手动禁用某个颜色
- 可通过滑条按颜色使用量逐步禁用颜色
- 导出的结果会和当前编辑状态保持一致

## 目录结构

```text
src/
  components/     界面组件
  data/           调色板和色系映射数据
  lib/            图像处理、i18n、主题等核心逻辑
  App.tsx         页面状态与主流程
```

## 数据来源

- `src/data/mard-palette-221.json`
- `src/data/color-system-mapping.json`

其中多色系映射数据来源于社区整理的拼豆色表映射。

## 当前适用场景

- 已有网格的拼豆图
- 放大后的像素图
- 带分隔线或缝隙的像素画
- 普通图片转拼豆图纸

## 注意事项

- 自动识别并不是对所有图片都百分之百准确
- 遇到复杂图片时，手动网格通常更稳定
- 浏览器端图像处理会占用一定内存，超大图片可能会比较慢
